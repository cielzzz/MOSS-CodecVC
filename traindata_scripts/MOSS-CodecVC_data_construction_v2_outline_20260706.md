## 0. 背景与动机

**现状**：no-text 训练三元组 = {source wav (说话人A), timbre wav (说话人B), target = Seed-VC(source→B)}。**target 本身是 Seed-VC 的输出** → 三重天花板：
1. **音色天花板**：模型学到的"像 B"最多 = Seed-VC 的"像 B"（而自研 VC 的立项目标恰恰是音色相似度超过 Seed-VC——用教师产物当 target 在目标上自相矛盾）；
2. **内容天花板**：Seed-VC 的错读/漏读被当成正确答案学进去；
3. **伪迹天花板**：教师的声学伪迹成为 target 分布的一部分。

**新思路（组会 2026-07-06）**：利用自有大规模爬取语料的**溯源信息**——同一条长音频裁出的两段 wav 天生同说话人。把教师从 target 侧挪到 source 侧：

```
同一长音频 → 裁出 u1, u2（同说话人 S，真实音频）
u1 --Seed-VC/其他VC--> u1'（内容韵律=u1，音色=随机他人）   ← 教师只污染输入侧
训练三元组：{ source = u1',  timbre_ref = u2,  target = u1 }
```

**本质**：这是机器翻译 back-translation 在 VC 上的对应物。教师缺陷从"必须模仿的答案"变成"输入侧的噪声/增广"；**target 上限解锁为真实音频**。任务定义完全对齐现有 no-text 模式：内容+韵律来自 source（u1' 韵律≈u1 韵律，VC 保持时序）、音色来自 timbre_ref（u2 音色=S=u1 音色）。✔ 自洽。

**免费红利**：text 模式数据同一批产出——{text=ASR(u1), source=u1', timbre_ref=u2, target=u1}，target 同样是真实音频（现有 text 数据同样受教师天花板限制）。

## 1. 核心构造流程（no-text 主线）

1. **同源对挖掘**：按溯源字段（同一长音频/同一 episode）配对裁剪段 → 候选 (u1, u2) 对；u1≠u2，内容不同（用 ASR 文本粗查重叠，避免近重复片段互当 ref/target）。
2. **伪 source 生成**：u1 → VC 转换到随机抽取的"扰动音色池"中的音色 → u1'。转换器首选 Seed-VC（现成管线），预留多转换器混合接口（见 §4 反教师指纹）。
3. **三元组落盘**：{u1', u2, u1} + 溯源 speaker id + 全部质检指标随行存储。

## 2. 与旧数据的关系

- 旧教师数据（166k 对）**保留为辅助**，不直接废弃；
- 消融设计（试点阶段就要做）：纯新 / 纯旧 / 混合（如 70:30）三臂，同规模同配方对比；
- 可选课程：先旧后新（教师数据当 warmup），视试点结果定。

## 3. 质检与过滤流水线（按序）

| # | 检查 | 方法/阈值（初值，试点后校准） | 防什么 |
|---|---|---|---|
| Q1 | 同说话人验证 | pair 内 ECAPA sim ≥ 阈值 + 长音频先过 diarization | 长音频多说话人（访谈/播客/换人）污染"天生同 speaker"假设 |
| Q2 | u1' 内容保真 | CER(ASR(u1'), ASR(u1)) ≤ ~0.05–0.10 | **教师错读进入 source 侧 → 模型学会"幻觉出 source 里没有的内容"**（比旧方案更危险，必须严查） |
| Q3 | 时长窗口 | u1 ∈ 2–15s，u2 ≥ 4s（timbre ref 要够 8s 推理窗用，短了记 bootstrap） | 分布对齐现有推理配置 |
| Q4 | 声学质量 | SNR/降噪残留/响度/截幅检查（沿用现有爬取清洗指标） | 脏 target 直接伤生成质量上限 |
| Q5 | 语种/配比 | zh/en 均衡，与 320 基准分布对齐 | 评测口径 |
| Q6 | u1' 音色漂移验证 | sim(u1', u1) ≤ 阈值（确认转换真的换掉了音色）+ sim(u1', 扰动目标) 抽查 | 转换失败的对=恒等映射训练样本，教模型"不转换" |

## 4. 反捷径设计（关键，逐条对应已知机制风险）

1. **信道泄漏**：u1/u2 同源录音共享信道与环境噪声特征 → 模型可能学"复制 ref 的信道指纹"冒充音色迁移，跨信道推理时失效或搬运噪声。缓解：
   - ref 侧（u2）随机信道扰动增广（编解码/EQ/混响，训练时随机开）；
   - 若说话人可跨长音频溯源（同一up主/主播多条长音频），**优先取不同录音的 u2**；
   - 验收留一个"跨信道 valid 子集"单独出数。
2. **反教师指纹**：模型可能学会"检测并撤销 Seed-VC 伪迹"而非通用音色转换（在真实人声 source 上失效）。缓解：
   - 转换器混合（Seed-VC + ≥1 个异构转换器，如 CosyVoice VC/Vevo/kNN-VC 类，比例待定）；
   - 扰动音色池要大且多样（性别/语种/年龄跨度），避免"少数几个假声"被记住；
   - 验收指标：真实跨说话人测试集（320 基准）上的增益必须 ≥ 合成 source 验证集上的增益趋势。
3. **timbre_ref 内容防抄**：u2 与 u1 内容不同（Q0 已保证）+ 沿用现有 ref-content F1 监控（当前配方全臂 F1≤0.07 的水位不能回退）。
4. **韵律不对齐风险**：若改用 TTS 重合成做 u1'（而非 VC），韵律时序会变 → **只允许保时序的转换器**进主线；TTS 重合成路线单独标注为"韵律弱监督"变体，不混入。

## 5. 规模与阶段

| 阶段 | 规模 | 动作 | 判据 |
|---|---|---|---|
| P0 盘点 | — | 语料总时长/说话人数/溯源字段覆盖率/可配对数量估算 | 可配对同源对 ≥ 100k 则主线可行 |

| P2 探针 | 新 vs 旧 vs 混合三臂，各同规模 | 3k-step LoRA 探针（沿用 recipe_final 配方与 quick20/320 评测） | binding rate / sim(gen,ref) / CER / fail 四指标；**成功线：新数据臂 sim 或 fail 显著优于旧数据臂** |

## 6. 预期收益与风险清单

- 预期：sim 上限解锁（target=真人真声）；CER 降（target 无教师错读）；论文叙事升级（"beyond the teacher ceiling"是审稿人必问项，dossier §6 已有披露伏笔）。
- 风险：①伪 source 分布与真实使用场景的 source 分布仍有 gap（毕竟是 VC 产物）——用多转换器+扰动池多样性压；②同源 pair 的信道捷径（§4.1）；③工程量：转换 100k+ 条的算力排期。
- **与当前模型迭代线的衔接**：数据 v2 与 recipe_v2（contiguous 短窗 C_ref 等）**正交可叠加**；数据准备可立即并行启动，但训练排期等 recipe_final 30k 收官判读后定（避免变量纠缠）。

## 7. 新会话启动清单（P0）

1. 盘点语料库：路径、总时长、长音频条数、溯源字段 schema、说话人可跨长音频关联性；
2. 抽 100 条长音频人工/diarization 抽查"单说话人假设"的真实成立率；
3. 选定转换器组合与扰动音色池来源；
4. 搭 P1 试点流水线（配对→转换→质检→落盘），复用现有 vcdata 清洗组件;
5. 出 P1 质检分布报告后回到主线程对齐（判据见 §5）。
